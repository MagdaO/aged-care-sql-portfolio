# Aged Care SQL Portfolio

## Overview
This project simulates a home care environment using SQL, including service rosters, client demographics, and data quality scenarios. It was thought to showcase practical SQL skills aligned with aged care and community services.

## Data Model
The project includes two main tables:

- **services**: records of home care services delivered, including staff, clients, time, duration, and cost
- **clients**: demographic information such as gender, country of birth, preferred language, and program type

## Key Skills 
- SQL table creation and data insertion
- Joins between operational and demographic data
- Aggregations for reporting (service usage, cost, workload)
- Data quality checks (missing demographics)
- Window functions (overlap detection in staff scheduling)

## Example Business Questions
- Which clients receive the most service hours?
- Which staff deliver the highest workload?
- Are there any overlapping staff schedules?
- Where are demographic data gaps impacting reporting?

## Tools Used
- Google BigQuery
- SQL

## Notes
This is a simulated dataset created for learning and portfolio purposes, based on real-world home care scenarios.
